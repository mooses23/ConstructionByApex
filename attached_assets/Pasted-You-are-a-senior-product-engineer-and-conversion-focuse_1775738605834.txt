You are a senior product engineer and conversion-focused web designer. Build a production-ready web app for a solo contractor brand called “Construction By Apex.”

CRITICAL RULES:
- This app must NOT be dependent on Replit-specific auth, hosting, database features, or runtime assumptions.
- The final app must be deployable on Vercel.
- Use Next.js App Router, TypeScript, Tailwind CSS, shadcn/ui, and Supabase.
- Keep architecture clean, modular, and simple.
- Do NOT import the old Apex Property Care architecture or feature bloat.
- This is a ground-up rebuild.
- The only legacy consideration is that Supabase may contain some old tables and RLS policies from v1, but the new app should be able to work from a fresh schema and ignore old baggage.
- Assume Supabase can be reset if needed.
- Focus on revenue, trust, usability, mobile conversion, local SEO, and clean admin workflow.
- Build for a solo contractor first, not a software company.

PROJECT GOAL:
Create a high-converting contractor website plus a lightweight admin dashboard for managing leads and quote requests.

BRAND:
Construction By Apex

TARGET MARKET:
Local residential and small commercial construction / repair / improvement customers in Ohio.

PRIMARY BUSINESS GOALS:
1. Generate more qualified leads
2. Increase calls, texts, and quote requests
3. Showcase trust and project quality
4. Keep lead management simple for one owner/operator
5. Make the brand look professional, modern, and credible

DO NOT BUILD:
- Enterprise CRM
- Opportunity finder
- RSS bidding system
- Complex invoice system
- Subscription billing
- Advanced role-based portals
- Replit auth
- Replit database dependencies
- Client self-service portal
- Complex automation engine
- E-signature integration in v1
- Overengineered analytics

BUILD THIS INSTEAD:
